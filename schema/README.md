# テーブル設計 / ER 図

このディレクトリには、ユーザー・組織・履歴（所属履歴/雇用履歴）を中心としたテーブル設計と ER 図（Mermaid）を配置します。

- `ER.mmd`: Mermaid 形式の ER 図

表示方法（例）:
- GitHub では Mermaid 対応のビューア拡張/プレビューを利用してください
- VSCode/Cursor の Mermaid プレビュー拡張でも表示できます

命名・前提:
- 物理テーブル名は `snake_case` を想定（ER 図上のエンティティ名も同様）
- `created_at` / `updated_at` は全テーブルで採用（監査のため）
- 一部 UI 上の自動計算/自動反映項目は、DB では「履歴テーブルの集計」で算出し、冗長保持はしない方針

今後の流れ:
1) 初期 ER 図をレビューし、未確定事項の合意を取る
2) 必要に応じて正規化/非正規化のバランスを調整
3) DDL（`schema.sql` など）を別途作成
